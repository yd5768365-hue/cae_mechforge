## 为什么开发 MechForge AI？

### 🎯 初衷

作为一名机械专业的学生，我在学习和实践中遇到了很多痛点：

| 痛点 | 描述 |
|------|------|
| **知识分散** | 机械设计手册、材料力学公式、有限元教程...资料太多太杂，查找困难 |
| **AI 幻觉** | 通用 AI 对机械专业知识理解不够，经常给出错误答案 |
| **工具割裂** | CAD 建模、网格划分、求解分析...每个环节都要切换不同软件 |
| **学习曲线** | CAE 软件学习成本高，界面复杂，新手难以上手 |

### 💡 解决方案

我决定开发一个**面向机械工程师的 AI 垂直助手**：

```
通用 AI + 机械知识库 + CAE 工具链 = MechForge AI
```

### 🚀 目标

> **真正懂机械、敢说真话、能真算**

- **真正懂机械**: 基于机械专业知识库，提供准确的专业解答
- **敢说真话**: 引用原文来源，杜绝 AI 幻觉
- **能真算**: 集成 CAE 工具链，从对话到计算一站式完成

## 项目理念

### 🎨 设计哲学

1. **简单至上**
   - 一键安装，开箱即用
   - 命令简洁，参数合理
   - 界面直观，学习成本低

2. **隐私优先**
   - 支持完全本地部署
   - 数据不上传云端
   - 用户掌控自己的知识库

3. **开放共享**
   - MIT 开源协议
   - 欢迎社区贡献
   - 文档齐全，易于二次开发

### 🌟 核心价值观

```
让 AI 真正服务于工程师，而不是替代工程师
```

- AI 是工具，不是答案机器
- 知识库提供参考，工程师做最终决策
- CAE 辅助验证，工程师理解原理
